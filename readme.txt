Данный веб-сервис представляет собой индивидуальный трекер привычек. Он позволяет пользователям регулярно отмечать выполнение заданных действий и визуально отслеживать свой прогресс изо дня в день для формирования новых или отказа от старых негативных привычек.
Целевой пользователь - любой человек, стремящийся улучшить свою дисциплину, внедрить в жизнь новые полезные привычки (например, «пить больше воды», «утренняя зарядка», «чтение») или избавиться от вредных, контролируя регулярность их выполнения.
1. Хранимые данные
В системе будут храниться три основные сущности:
  Пользователи: базовая информация (имя/логин, email, пароль);
  Привычки: название привычки, её описание, дата добавления, тип (бинарная или количественная);
  Отметки о выполнении (логи): записи, фиксирующие факт выполнения конкретной привычки в определенную дату.
2. Основные действия (Функционал)
Пользователь сможет выполнять следующие действия:
  Просматривать список всех своих привычек;
  Создавать новые привычки;
  Редактировать существующие привычки (изменять название или описание);
  Удалять неактуальные привычки;
  Отмечать привычку как «выполненную» в текущий или выбранный день;
  Просматривать историю выполнения (статистику прогресса) по каждой привычке за разные временные интервалы;
  Получать общую статистику по всем привычкам за выбранный временной интервал (неделю, месяц(.
3. Структура базы данных
Таблица users (Пользователи)
  id (INTEGER, Primary Key) - уникальный идентификатор;
  username (VARCHAR) - имя пользователя;
  email (VARCHAR) - электронная почта;
  password (VARCHAR) - пароль пользователя;
  created_at (DATETIME) - дата регистрации пользователя.
Таблица habits (Привычки - основная сущность)
  id (INTEGER, Primary Key) - уникальный идентификатор привычки;;
  title (VARCHAR) - краткое название привычки (например, "Бег по утрам");
  description (VARCHAR) - подробное описание;
  created_at (DATETIME) - дата создания привычки;
  user_id (INTEGER, Foreign Key) - связь с пользователем (создатель привычки);
  type (VARCHAR) - тип привычки.
Таблица habit_logs (Отметки о выполнении)
  id (INTEGER, Primary Key) - уникальный идентификатор записи.
  completed_date (DATE) - дата, когда привычка была выполнена.
  habit_id (INTEGER, Foreign Key) - связь с привычкой (указание, какая именно привычка была выполнена).
